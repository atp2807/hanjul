## 0. 크리티컬 패스 (문 여는 최소 순서)

병렬로 시작하되, 아래 의존성이 실제 순서를 강제한다.

```
[지금 즉시·병렬]
 A. 상표 출원(개인명의)  ─ 사업자등록 안 기다림. 빠를수록 좋음.
 B. 도메인/핸들 방어

[행정 체인 — 순차 의존]
 사업자등록 ──▶ 사업용 계좌 ──▶ PG 계약(단순결제) ──▶ 에스크로 확인증 ──▶ 통신판매업 신고
      └──▶ 출판사 신고 ──▶ 발행자 등록(seoji) ──▶ ISBN 발급

[코드 게이트 — 위와 병렬로 개발, 값은 행정에서 주입]
 실결제 ON  ·  법률문서+사업자정보 푸터  ·  정산/출금  ·  회원탈퇴

문 여는 최소 = 사업자등록 + 통신판매업 신고 + PG 실연동 + 법률문서 게시 + 사업자정보 표시
```

**★ 결제 대원칙(계약 전 반드시)**: PG는 **단순결제(대표가맹점)**로 계약. **정산대행/에스크로대행/전자금융업 아님.** 한줄은 "정산대행"이 아니라 **"정산"** — 플랫폼이 판매자이고 작가에게 **인세(royalty)** 를 지급하는 구조. (전자금융업 등록 회피 = 이 구조 유지가 핵심)

## 3. 행정 등록  🔴

순서 핵심: **사업자등록 → 사업용 계좌 → (PG→에스크로확인증)→통신판매업 신고** / **출판사 신고 → 발행자 등록**(병렬 가능하나 발행자 등록은 출판사 신고 후).

### 사업 구조 (선행 결정)
- [ ] 개인사업자 vs 법인 — **혼자면 개인사업자로 시작**(후 법인전환 흔함) 권장
- [ ] (법인 시) 설립등기 — 정관·자본금·등기

### 사업자등록
- [ ] 사업장 주소 확보(자택 가능 / 임대 시 가능여부)
- [ ] 업종코드 다수 등록 — 전자상거래 소매, 출판업, 정보서비스(플랫폼), 광고대행 등
- [ ] 사업자등록 신청(홈택스/세무서) → 사업자등록증 수령
- [ ] 사업용 계좌 개설 · 사업용 카드 발급

### 출판 관련
- [ ] 출판사 신고(관할 시·군·구청) → **출판사신고확인증**
- [ ] seoji(서지정보유통지원시스템) 발행자 등록 → **발행자번호**
- [ ] ISBN 신청 절차 숙지(발행자번호 후 책별 개별 발급)
- [ ] 서지정보 등록 절차(※ CIP 폐지 → ISBN 신청 시 서지정보 제출로 대체)
- [ ] 납본 절차(국립중앙도서관 + 국회도서관, 발행 후 의무)

### 쇼핑몰(전자상거래)
- [ ] **구매안전서비스(에스크로) 가입확인증** 확보(PG/은행 통해) — 통신판매 신고 선결
- [ ] 통신판매업 신고(관할 구청) → **통신판매번호**(푸터 표시용)
