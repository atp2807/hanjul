# PG 계약 미팅팩 (2026-07-08)

> 목적: PG(결제대행사) 계약 미팅에서 물어볼 것 + 후보 비교. 최종 계약조건은 미팅 후 확정.
> 대전제(양보 불가): **단순결제(대표가맹점) 계약** — 정산대행/에스크로대행 아님. `docs/payment-tax.md §1` 참조.

## 반드시 물어볼 질문 (우선순위순)

1. **"저희처럼 여러 작가 콘텐츠를 자사 명의로 판매하고, 판매자(사)가 작가에게 별도 인세(로열티)를 지급하는 구조인데, 이게 마켓플레이스/정산대행으로 분류되나요, 단순 대표가맹점 결제로 가능한가요?"** — 가장 중요한 질문, 답변에 따라 계약형태가 갈림.
2. "구매안전서비스(에스크로) 확인증 발급 절차와 소요기간은?" (통신판매업 신고는 이미 완료 — 확인증만 소급 필요할 수 있음)
3. "전자책(디지털콘텐츠) 결제 지원되나요? 실물 배송 전제 조건 있나요?"
4. "정산 주기, PG 수수료율, 결제수단별(카드/간편결제) 수수료 차이는?" — ★**결정: 카드 + 간편결제만 받는다. 계좌이체·가상계좌는 안 켠다**(현금성 수단 최소화). 상점관리자에서 계좌이체·가상계좌 비활성 요청.
5. ★**현금영수증 — 카드전용이면 불필요(카드전표=적격증빙)**. 단 간편결제는 자금원이 갈림: 카드연동분=카드취급(현금영수증X), **머니·계좌출금형 간편결제(토스머니·네이버/카카오페이 머니)=현금성 → 현금영수증 대상**. 질문: **"결제위젯에서 이 현금성 간편결제 건의 현금영수증을, 위젯이 정보 수집해서 PG(토스)가 발급 대행하나요, 아니면 가맹점(저희)이 발급해야 하나요?"** — PG 대행이면 한줄 코드 0줄, 가맹점 몫이면 토스 현금영수증 API 붙임(라이브 전환 시 결정). (`docs/payment-tax.md §4`)
6. "가맹점 심사 기간과 필요 서류(사업자등록증·통신판매업신고증·대표자 신분증 등)는?"
7. "웹훅(결제완료 통보) 지원 방식과 재시도 정책은?" (이미 Toss 연동 코드 있음 — 계약 PG가 Toss면 그대로, 아니면 어댑터 신규 필요)

## 후보 비교 (초안 — 실제 조건은 미팅으로 확정)

| | 토스페이먼츠 | 포트원(구 아임포트) | 나이스페이 |
|---|---|---|---|
| 이미 연동 코드 | ✅ (`backend/src/features/billing/infrastructure/toss_client.py`) | 포트 설계만(`PaymentGateway` 어댑터 자리 있음) | 없음 |
| 디지털콘텐츠 지원 | 실사용 확인 필요 | 실사용 확인 필요 | 실사용 확인 필요 |
| 대표가맹점 상품 | 확인 필요 | 확인 필요(정산대행 상품이 주력이라 단순결제 옵션 별도 문의) | 확인 필요 |

> 이미 토스 샌드박스 연동이 코드로 존재(`lr-cec34d0b`)해서, **토스 먼저 미팅**하는 게 계약~라이브 전환 속도상 유리. 단, 질문 1의 답이 "마켓플레이스로 분류됨"이면 재검토.

## 계약 후 즉시 할 일
- `.env`의 `TOSS_TEST_*` → 라이브 키로 교체
- `PAYMENT_DEMO=False` (이미 코드 기본값, `.env`만 바꾸면 됨)
- 에스크로 확인증 최종본 사업자정보에 반영
