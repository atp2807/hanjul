# 결제·세금 실무 (한줄 정산 구조) — 조사 2026-07-01

> ⚠️ 아래는 WebSearch 기반 실무 정리. **핀테크/전금법 로펌 + 세무사 최종확인 필수**(표시된 항목). 시행령 조문·세율은 적용 시점 원문 대조.

## 0. 한 줄 결론
플랫폼=판매자(대표가맹점), 대금을 **자기 매출**로 인식, 작가에 **인세(로열티) 별도 지급** = **전자금융업(PG) 등록 회피 근거 있음**. **ISBN/ECN 확보 시 전자책 부가세 면세**. 작가 인세는 **사업소득 3.3%** 디폴트.

## 1. PG 연동 + 전자금융업 회피 선 ★
- 연동: `주문 사전저장(서버가 금액 확정) → 결제창 → 서버 confirm(금액검증) → 웹훅 재조회`. 프론트 콜백 불신, 서버 confirm+웹훅 이중화. → **한줄은 이미 이 구조**(주문금액 서버 도출 + 웹훅 reconcile).
- **금융위 기준(2024.6)**: "**타인(판매자) 판매대금 정산을 대행·매개**"하면 PG 등록의무(자본금 10억·미등록 형사처벌). 회피 = ①외부 PG가 정산대행 ②중개 등 **부수적** 정산 ③자금관여 없이 정보만.
- **한줄 위치**: 자기명의 판매→**자기 매출**이라 애초에 "타인 대금"이 아님. 인세는 "정산대행"이 아니라 **로열티 채무 이행** → PG 개념징표 비해당.
- **유지 조건(지켜야 성립)**:
  1. 약관·정산약정·세무에서 **"한줄=판매자, 작가=콘텐츠 공급자(라이선서)"** 일관 문서화. 영수증도 한줄 명의.
  2. 인세를 "판매액 X% 대신 받아 정산"❌ → **"계약상 로열티 지급"**⭕. "정산 대행/매개" 표현 회피.
  3. **오픈마켓/중개형 금지** — "작가가 가격 정하고 한줄은 자리만, 대금 받아 넘김"이 되면 PG 대상 + (2026.12) 정산자금 외부관리까지. ← **선 넘음 지점**.
  4. 에스크로·캐시충전(선불)·후불(BNPL) 붙이면 각각 별도 규제 → 별도 검토.
- 동향: PG 정산자금 외부관리 의무화(2026.12 시행)는 **PG업자 대상 → 한줄 비대상**(회피 이유 하나 더). 선불충전금 100% 별도관리는 캐시 발행 안 하면 무관.
- **⚠️ 법무확인**: 형식만 판매자·실질 중개면 실질과세/규제로 뒤집힐 수 있음.

## 2. 전자책 부가세 면세 ★
- 근거: 부가세법 §26①8 → 시행령 §38① → 시행규칙 §26 → **문체부 고시(2024.12.11 개정)**.
- **면세 요건**: ①출판업자 발행 ②기록사항(저자·발행인·발행일·정가) ③**ISBN(/ISSN) 또는 KEPA ECN** 부여. 개정으로 오디오북·멀티미디어북까지 확대.
- **과세(10%)**: ISBN/ECN 없는 단순 PDF·문서(크몽/스마트스토어형), 요건 미달, 실질이 음악/영상/게임.
- **혼재 처리**: 면세→(면세)계산서 / 과세→세금계산서(면세는 세금계산서 불가). B2C는 카드전표·현금영수증 갈음, B2B 요청 시 계산서. 공통 매입세액은 과세/면세 비율 **안분**(면세분 불공제).
- **⚠️ 세무확인**: 면세 귀속주체(출판업자 공급 vs 중개), 저작권료 vs 출판물 공급대가 구분(국세청 계약서 문구 중시).

## 3. 원천징수 (플랫폼 → 작가 인세)
| 케이스 | 구분 | 원천징수율 |
|---|---|---|
| 저작자 본인·계속반복 창작 인세 | **사업소득** | **3.3%** |
| 저작자 본인·일시적 원고료 | 기타소득(§21①15) | 실효 **8.8%**(필요경비 60% 의제 후 22%) |
| 저작자 외의 자(상속·양수) 저작권료 | 기타소득(§21①5) | 필요경비 없음 → ~22% |
- **디폴트 = 사업소득 3.3%** (셀프퍼블=계속반복 등록·판매). 일시 기고자도 실무상 사업소득 일괄이 안전.
- 필요경비 의제는 **현행 60%**(과거 80/90%는 개정 전).
- **지급명세서**: 사업소득=**간이지급명세서 매월**(다음달 말). 기타소득 원고료(§21①15)=연 지급명세서(2월 말), 세액 0도 제출. **원천징수영수증 교부** 의무.
- 권고: 사업소득 3.3% + 매월 간이지급명세서 데이터 + 영수증 교부. 양수/유족은 P2.

## 4. 증빙 (구매자)
- 전자상거래 소매/통신판매 = **현금영수증 의무발행 업종**. 건당 **10만원↑ 현금거래는 요청 없어도 의무발행**(인적사항 모르면 자진발급 010-000-1234). 미발급 가산세 20%.
- 전자책은 대개 10만↓지만 **요청 시 1원부터 의무** → 자동발급 로직 필수.
- 카드결제는 카드전표가 증빙(현금영수증 대상 아님). 현금영수증은 현금/이체만.
- **⚠️ PG 경유 시 현금영수증 발급 주체** 확인.

## 코드 반영 (→ settlement-schema.md 참조)
- `pub.book`: ISBN(있음) + **ECN 필드 + tax_exempt 플래그** → 부가세·증빙 분기.
- `bill`: 소비자 결제=카드전표/현금영수증+면세과세 분기 / 작가 정산=사업소득 3.3% 원천징수 + 간이지급명세서.
